## 2. Теоретическая часть

### 2.1. Что такое триангуляция?

**Триангуляция** — это разбиение множества точек на плоскости на треугольники таким образом, чтобы:
- Вершинами треугольников были только точки из исходного множества
- Треугольники не пересекались (кроме общих рёбер и вершин)
- Вся выпуклая оболочка точек была покрыта треугольниками

### 2.2. Жадный алгоритм триангуляции

Жадный алгоритм — это эвристический подход, который на каждом шаге выбирает локально оптимальное решение. В контексте триангуляции:

**Принцип работы:**
1. Генерируются все возможные рёбра между точками
2. Рёбра сортируются по длине (от короткого к длинному)
3. Рёбра добавляются последовательно, только если они не пересекают уже добавленные
4. Из добавленных рёбер формируются треугольники

**Преимущества:**
- Простота реализации
- Быстрая работа на небольших и средних множествах точек
- Визуально приятные результаты (предпочтение коротким рёбрам)

**Недостатки:**
- Не гарантирует триангуляцию Делоне
- Может не обеспечить полное покрытие в некоторых случаях
- Сложность O(n² log n), где n — количество точек

## 3. Математическая часть: Формулы и вычисления

### 3.1. Расстояние между точками

Для сортировки рёбер используется евклидово расстояние:

```
d(A, B) = √[(x₂ - x₁)² + (y₂ - y₁)²]
```

**Реализация в коде:**
```java
private static double dist(Point a, Point b) {
    double dx = a.x - b.x, dy = a.y - b.y;
    return Math.sqrt(dx * dx + dy * dy);
}
```

### 3.2. Проверка пересечения отрезков

Для проверки пересечения двух отрезков AB и CD используется **векторное произведение**.

**Векторное произведение** (cross product):
```
cross(A, B, C) = (B.x - A.x) × (C.y - A.y) - (B.y - A.y) × (C.x - A.x)
```

Это значение определяет положение точки C относительно прямой AB:
- **> 0**: C слева от AB
- **< 0**: C справа от AB  
- **= 0**: C лежит на прямой AB

**Условие пересечения отрезков:**

Отрезки AB и CD пересекаются, если:
1. Точки C и D лежат **по разные стороны** от прямой AB
2. Точки A и B лежат **по разные стороны** от прямой CD

```java
private static boolean segmentsIntersect(Point p1, Point p2, Point p3, Point p4) {
    double d1 = cross(p3, p4, p1);  // Положение p1 относительно p3-p4
    double d2 = cross(p3, p4, p2);  // Положение p2 относительно p3-p4
    double d3 = cross(p1, p2, p3);  // Положение p3 относительно p1-p2
    double d4 = cross(p1, p2, p4);  // Положение p4 относительно p1-p2
    return ((d1 > 0 && d2 < 0) || (d1 < 0 && d2 > 0)) &&
           ((d3 > 0 && d4 < 0) || (d3 < 0 && d4 > 0));
}
```

### 3.3. Проверка точки внутри треугольника

Используется метод **барицентрических координат** через векторные произведения.

Точка P находится внутри треугольника ABC, если:
- P с одной стороны от всех трёх рёбер (AB, BC, CA)
- Все три векторных произведения имеют одинаковый знак

```java
private static boolean pointInTriangle(Point p, Point a, Point b, Point c) {
    double d1 = cross(a, b, p);
    double d2 = cross(b, c, p);
    double d3 = cross(c, a, p);
    
    // Проверяем, что точка не на границе (с учётом погрешности)
    boolean hasNeg = (d1 < -1e-9) || (d2 < -1e-9) || (d3 < -1e-9);
    boolean hasPos = (d1 >  1e-9) || (d2 >  1e-9) || (d3 >  1e-9);
    
    return !(hasNeg && hasPos)
           && Math.abs(d1) > 1e-9 && Math.abs(d2) > 1e-9 && Math.abs(d3) > 1e-9;
}
```

**Примечание:** Используется погрешность `1e-9` для учёта ошибок вычислений с плавающей точкой.

## 4. Алгоритм: Пошаговое описание

### Этап 1: Подготовка данных

1. **Удаление дубликатов точек:**
   - Проверяются все точки на совпадение координат
   - Точки с разницей координат < 1e-9 считаются дубликатами
   - Оставляется только уникальное множество

```java
List<Point> unique = new ArrayList<>();
for (Point p : points) {
    boolean dup = false;
    for (Point u : unique) {
        if (Math.abs(p.x - u.x) < 1e-9 && Math.abs(p.y - u.y) < 1e-9) {
            dup = true;
            break;
        }
    }
    if (!dup) unique.add(p);
}
```

### Этап 2: Генерация всех возможных рёбер

Для n точек генерируется **C(n,2) = n(n-1)/2** рёбер:

```java
List<int[]> allEdges = new ArrayList<>();
for (int i = 0; i < n; i++)
    for (int j = i + 1; j < n; j++)
        allEdges.add(new int[]{i, j});
```

### Этап 3: Сортировка рёбер по длине

```java
allEdges.sort((a, b) -> {
    double da = dist(unique.get(a[0]), unique.get(a[1]));
    double db = dist(unique.get(b[0]), unique.get(b[1]));
    return Double.compare(da, db);
});
```

**Сложность:** O(n² log n)

### Этап 4: Жадный отбор рёбер

Проходим по отсортированным рёбрам и добавляем ребро, только если оно:
- Не пересекает уже добавленные рёбра
- Не имеет общих вершин с проверяемым ребром (при проверке пересечений)

```java
List<int[]> accepted = new ArrayList<>();
for (int[] edge : allEdges) {
    if (!intersectsAny(edge, accepted, unique))
        accepted.add(edge);
}
```

**Сложность проверки одного ребра:** O(m), где m — количество уже добавленных рёбер  
**Общая сложность этапа:** O(n² × m) ≈ O(n³) в худшем случае

### Этап 5: Формирование треугольников

Ищем все комбинации трёх точек (i, j, k), для которых:
1. Все три ребра (i-j, i-k, j-k) присутствуют в `accepted`
2. Внутри треугольника нет других точек

```java
for (int i = 0; i < n; i++) {
    for (int j = i + 1; j < n; j++) {
        if (!edgeSet.contains(edgeKey(i, j))) continue;
        for (int k = j + 1; k < n; k++) {
            if (!edgeSet.contains(edgeKey(i, k)) || 
                !edgeSet.contains(edgeKey(j, k))) continue;
            
            Point pi = unique.get(i);
            Point pj = unique.get(j);
            Point pk = unique.get(k);
            
            // Проверяем, что внутри нет других точек
            boolean empty = true;
            for (int m = 0; m < n; m++) {
                if (m == i || m == j || m == k) continue;
                if (pointInTriangle(unique.get(m), pi, pj, pk)) {
                    empty = false;
                    break;
                }
            }
            if (empty) triangles.add(new Triangle(pi, pj, pk));
        }
    }
}
```

**Сложность:** O(n⁴) — перебор троек точек и проверка остальных точек

## 7. База данных SQLite

### 7.1. Схема базы данных

#### Таблица `sessions`
Хранит информацию о сохранённых сессиях триангуляции.

| Поле           | Тип      | Описание                                    |
|----------------|----------|---------------------------------------------|
| id             | INTEGER  | Первичный ключ, автоинкремент              |
| name           | TEXT     | Название сессии                             |
| point_count    | INTEGER  | Количество точек в сессии                   |
| triangle_count | INTEGER  | Количество треугольников в сессии           |
| created_at     | TEXT     | Дата и время создания (локальное время)     |

#### Таблица `points`
Хранит точки для каждой сессии.

| Поле       | Тип      | Описание                                        |
|------------|----------|-------------------------------------------------|
| id         | INTEGER  | Первичный ключ, автоинкремент                  |
| session_id | INTEGER  | Внешний ключ на sessions.id                     |
| idx        | INTEGER  | Порядковый номер точки в сессии                 |
| x          | REAL     | Координата X                                    |
| y          | REAL     | Координата Y                                    |

#### Таблица `triangles`
Хранит треугольники для каждой сессии.

| Поле       | Тип      | Описание                                        |
|------------|----------|-------------------------------------------------|
| id         | INTEGER  | Первичный ключ, автоинкремент                  |
| session_id | INTEGER  | Внешний ключ на sessions.id                     |
| idx        | INTEGER  | Порядковый номер треугольника в сессии          |
| p1x, p1y   | REAL     | Координаты первой вершины                       |
| p2x, p2y   | REAL     | Координаты второй вершины                       |
| p3x, p3y   | REAL     | Координаты третьей вершины                      |

### 7.2. Операции с базой данных

**Сохранение сессии:**
1. Создаётся запись в таблице `sessions`
2. Сохраняются все точки в таблице `points`
3. Сохраняются все треугольники в таблице `triangles`
4. Всё выполняется в одной транзакции (ACID)

**Загрузка сессии:**
1. Выбирается сессия по id
2. Загружаются точки, отсортированные по idx
3. Загружаются треугольники, отсортированные по idx

**Удаление сессии:**
- Благодаря `ON DELETE CASCADE` при удалении сессии автоматически удаляются связанные точки и треугольники

## 8. Пользовательский интерфейс

### 8.1. Основные режимы работы

#### Режим рисования (активен по умолчанию)
- **ЛКМ** (левая кнопка мыши) — добавить точку
- **ПКМ** (правая кнопка мыши) — удалить последнюю точку
- Координаты точек отображаются в реальном времени
- Автоматическая триангуляция при добавлении каждой точки (опционально)

#### Режим ввода координат
- Отключен режим рисования
- Точки вводятся вручную в текстовом поле
- Формат: `x y` (одна пара координат на строку)
- Строки с `#` игнорируются как комментарии

### 8.2. Функции приложения

| Функция                    | Описание                                                     |
|----------------------------|--------------------------------------------------------------|
| **Триангулировать**        | Запуск жадного алгоритма триангуляции                        |
| **Очистить**               | Удаление всех точек и треугольников                          |
| **Случайные точки**        | Генерация 10-20 случайных точек                              |
| **Загрузить из файла**     | Загрузка точек из текстового файла                           |
| **Сохранить в файл**       | Сохранение точек в текстовый файл                            |
| **Сохранить в БД**         | Сохранение текущей сессии в базу данных                      |
| **Загрузить из БД**        | Выбор и загрузка сохранённой сессии                          |
| **Авто-триангуляция**      | Включение/выключение автоматической триангуляции             |

### 8.3. Визуализация

**Отображение на холсте:**
- **Точки** — красные круги с белой обводкой
- **Рёбра** — синие линии
- **Координаты точек** — подписи рядом с точками (опционально)

**Автоматическое масштабирование:**
- Холст автоматически подстраивается под размер окна
- Точки масштабируются с учётом их диапазона координат
- Сохраняются отступы от краёв холста

## 11. Сложность алгоритма

### Временная сложность

| Этап                          | Сложность       | Описание                                    |
|-------------------------------|-----------------|---------------------------------------------|
| Удаление дубликатов           | O(n²)           | Сравнение каждой точки с каждой             |
| Генерация рёбер               | O(n²)           | Генерация всех пар точек                    |
| Сортировка рёбер              | O(n² log n)     | Сортировка n² рёбер                         |
| Отбор рёбер (жадный)          | O(n³)           | Проверка пересечения для каждого ребра      |
| Формирование треугольников    | O(n⁴)           | Перебор троек и проверка остальных точек    |
| **Общая сложность**           | **O(n⁴)**       | Доминирует формирование треугольников       |

### Пространственная сложность

- **O(n²)** — хранение всех возможных рёбер
- **O(n)** — хранение точек
- **O(t)** — хранение треугольников (t ≤ 2n - 5 по теореме Эйлера)

**Общая:** O(n²)
